System Design

Two Independent Tracking Layers

Geofencing handles arrival. GPS tracking handles the shift. Different tools, different battery costs.

🔔
Layer 1 — Native Geofencing
iOS CoreLocation / Android Geofencing API monitors a set of circular regions at the OS level. The OS wakes your app when a boundary is crossed — even if the app is killed.
✓ Works when app is killed
✓ ~1–2% extra battery/day
✓ Uses hardware motion coprocessor
! iOS limit: 20 regions per app
Used for: Clock-in prompt when tech arrives at job site
📡
Layer 2 — Active GPS Tracking
Continuous GPS tracking via expo-location every 30 seconds while clocked in. Powers the live crew map, GPS trail history, drive detection, and auto cost code allocation.
✓ Full trail: driving vs. on-site
✓ Real-time dispatch map updates
✓ Stops completely on clock-out
! ~8–12% extra battery on active shift
Used for: Live map, route history, labor cost allocation
